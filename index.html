HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • HTML attributes are always defined in the beginning element.
  • The title defines the title for the webpage which is very important for SEO.
  • Lang = "en" tells the browser the language for the webpage. "En" stands for English.
  • Child elements are ones that are nested inside other elements.
  • Parest elements are ones that hold all the children.
  • Semantic HTML is the name of elements that describe its content and role it plays on a webpage.
  • Not all elements are semantic.
  • It's best practice to use semantic elements wherever possible.
  • We insert the contect between an opening tag and a closing tag so the browser understands how to treat content.
  • Closing tags begin with a /.
  • An element refers to an entire element.
  • A tag refers to only what appears inside the angle brackets.
  • The words 'element' and 'tag' are commonly used interchangeably.
  • Alt text is important for screenreaders and other assistive technologies. It can improve search engine rankings.
  • Developers text code early to often catch bugs. Sometimes this is as siple as looking for changes in the web browser.

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git must precede every git statement
  • git checkout -b feature/"": creates a new feature branch
  • git add -A: indicates we want to make changes
  • git commit -m"": describes what the commit contains
  • Do not code in the main or master branch!
  • Pull requests are required to make changes to the main branch
  • Always remember to stage files after saving any changes
